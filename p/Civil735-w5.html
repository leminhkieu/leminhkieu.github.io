Principles of Prompt Engineering

1. Be Clear and Specific

2. Provide Context

3. Use Technical Terminology Accurately

4. Structure Requests According to the Coding Task

1. Be Clear and Specific

Bad prompts:

“Generate code for a simulation.”

Good prompts:

“I am developing a traffic flow simulation in Python using discrete-event simulation principles. Please write a Python class called IntersectionSimulator that:

2. Provide Context

Bad prompts:

“Write code to get data out of a file with pandas or something.”

Good prompts:

“In Python, write a function called calculate_stats that takes a list of integers and returns a dictionary containing the minimum value, maximum value, average, and standard deviation of the list.”

3. Use Technical Terminology Accurately

Bad prompts:

“Write me some code to do calculations.”

Good prompts:

“Using the pandas library in Python, write a function read_and_clean_csv that: 1. Loads data from a CSV file called ‘data.csv’. 2. Drops any rows with missing values. 3. Converts columns ‘time’ and ‘date’ to appropriate datetime formats. 4. Returns a cleaned DataFrame ready for further analysis. Make sure to handle potential file I/O errors gracefully.”

4. Structure Requests According to the Coding Task

• For debugging: describe the error, show problematic code, and ask for an explanation plus a fix.

• For optimization: define performance expectations, highlight known issues or complexities, and request specific improvements.

• For documentation: specify the style (docstring vs. comment block) and the details to include (parameters, return values, usage).
